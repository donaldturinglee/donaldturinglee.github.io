Docker笔记

Donald Lee2023-12-162023-12-22

当前用户不在docker组中时，请添加sudo

容器管理

docker system info : 查看系统的一些相关信息
docker –help : 查看相关命令的详细说明

命令的选项有四种： 长选项、短选项、复合选项、无选项

命令 描述
docker container attach
docker image build
docker container commit
docker container cp
docker container create
docker stack deploy
docker container exec
docker image ls
docker image import
docker system info
docker container/image inspect
docker container kill
docker container port
docker container ls 查看容器
docker container restart
docker container rm
docker image rm
docker container run 在指定的镜像IMAGE上创建一个可写的容器
docker container start
docker container top

一些常用的配置项

  • -i或–interactive 交互模式
  • -t或–tty 分配一个伪终端
  • -d或–detach 在后台运行容器并输出容器的ID到终端
  • –rm 在容器退出后自动移除
  • -p 将容器的端口映射到主机
  • v或–volume 指定数据卷

两种退出bash的方式

  • 使用exit命令： bash终止，容器进入到停止状态
  • 使用组合键： 使用组合键退出，容器仍然保持运行的状态，可以再次连接到这个bash中。组合键是ctrl + p 和 ctrl + q.

docker run 命令的作用并不是创建一个容器，而是在一个新的容器中运行一个命令。

docker container create 创建一个新容器
常见配置项

  • –name 指定一个容器名称，未指定时，会随机产生一个名称
  • –hostname 设置容器的主机名
  • –mac-address 设置MAC地址
  • –ulimit 设置ulimit选项

ulimit 对容器运行时的一些资源进行限制

docker container start 启动容器

使用 docker container create 运行一个容器时，分成两步，即创建和启动
使用 docker container run 只需要一步，run 命令同时完成了 create 及 start 操作

docker container stop 停止容器
docker container restart 重启容器
docker container pause 暂停容器中进程
docker container unpause 恢复容器中进程

docker container ls 查看容器列表
可选配置项

  • -a 显示所有的容器
  • -q 仅显示ID
  • -s 显示总的文件大小
    默认情况下，仅显示正在运行的容器

docker container attach 将本地标准输入输出流连接到一个运行中的容器

docker container inspect 查看容器的详细信息（元数据）

docker container logs 获取容器的输出信息
常用配置项

  • -t或–timestamps 显示时间戳
  • -f 实时输出，类似于tail -f

docker container top 显示运行中的容器的进程信息（对于并未运行的容器是无效的）

docker container diff 显示相对于镜像的文件系统来说，容器中做了什么改变
两种改变：系统运行过程中变更或更新的; 我们创建的

docker container exec 在一个运行中的容器中执行命令

docker container rm 删除容器（在删除容器后，在容器中进行的操作不会持久化到镜像中）
docker container rm -f $(docker container ls -aq) 删除创建的所有容器
$(docker container ls -aq) 会输出所有容器的UUI, rm命令可以根据UUID去删除容器, -f 选项是因为还有运行中的容器, 所以需要强制删除

docker container prune 删除容器, docker 1.13及以上版本支持
常用配置项

  • -f或–force 跳过警告提示
  • –filter执行过滤删除

镜像管理

archlinux: latest : 镜像名称, 冒号前面的 archlinux 是仓库名, 后面的 latest 是TAG, 不同的TAG可以对应相同的镜像, TAG通常设置为镜像的版本号. 在镜像后没有添加TAG的情况下, Docker会使用默认的TAG: latest
Docker的镜像是分层存储,每一个镜像都是由很多层组成的.而一些镜像会共享一些相同的层
使用 docker system info | grep “Storage Driver” 查看使用的存储驱动

docker image ls 查看镜像
docker image inspect 查看镜像的详细信息
docker search 显示Docker Hub中所有包含archlinux关键字的镜像

docker image pull 从Docker Hub中获取镜像或仓库
常用配置项

  • -a 下载仓库中的所有镜像,即下载整个存储库

docker container commit 在容器中进行修改,然后将修改提交到一个新的镜像中
详细解释: 从一个容器的修改中创建一个新的镜像. 例如, 运行一个容器, 然后中其中创建一个文件, 最后使用 commit 命令
在生产环境中使用Dockerfile是更好的选择

docker image build 从一个Dockerfile文件中读取指令构建一个新的镜像
常用配置项

  • -t 指定新的镜像, . 是基于该目录下的Dockerfile构建
    Dockerfile: 一个包含用户构建镜像命令的文本文件

Dockerfile一般包含以下几部分

  • 基础镜像: 使用 FROM 命令指定基础镜像, 一个Dockerfile必须以 FROM 指令启动

–restart=always 选项保证容器始终不关闭

docker image rm 删除镜像(如果该镜像正被一个容器所使用,需要将容器删除后才能成功删除镜像)

数据保存在Docker中的一些缺点:

  • 当容器不再运行时, 无法使用数据, 并且容器被删除时, 数据也随之被删除, 并不会被保存
  • 数据保存在容器中的可写层中, 无法轻松的将数据移动到其他地方

针对以上的缺点, Docker提供三种不同的方式将数据从Docker主机中挂载到容器中

  • 卷(volumes): 存储在Docker管理的主机文件系统的一部分中(/var/lib/docker/volumes/)中,完全由Docker管理
  • 绑定挂载(bind mounts): 可以将主机上的文件或目录挂载到容器中
  • 临时文件系统(tmpfs): 仅存储中主机系统的内存中,而不会写入主机的文件系统

上述的三种方法中,数据中容器内看上去都是一样的. 它被认为容器文件系统中的目录或单个文件.
大多数情况下 volumes 总是正确的(它完全由Docker管理, 更容易备份或迁移, 并且可以使用 Docker CLI 命令来管理 volumes )

docker volume ls 列出本地可用的 volumes 列表
docker volume create 创建卷
由系统随机生成名称的创建卷被称为 匿名卷 , 直接使用该卷需要指定卷名, 即自动生成的 ID.
创建卷时一般手动指定 name, 以下是一个示例

在使用卷启动一个容器前的常用配置项

  • -v或–volume
    • 由三个冒号(:)分割的字段组成, [HOST-DIR:]CONTAINER-DIR[:OPTIONS]
    • HOST-DIR 主机上的目录或数据卷的名字. 省略该部分时, 会自动创建一个匿名卷. 如果是指定主机上的目录, 需要使用绝对路径
    • CONTAINER-DIR 将要挂载到容器中的目录或文件, 即表现为容器中的某个目录或文件
    • OPTIONS 代表配置, 例如设置为只读权限( ro ), 此卷仅能被该容器使用( Z ), 或者可以被多个容器使用 ( z ), 多个配置项使用逗号分隔

例: -v volume1:/volume1:ro, z

  • –mount
    • 由多个键值对组成, 键值对之间由逗号分隔. 例: type=volume, source=volume1, destination=/volume1,ro=true
    • type 指定类型, 可以指定为bind, volume tmpfs
    • source 当类型为volume时, 指定卷名称, 匿名卷时省略该字段. 当类型为bind时, 指定路径, 可以使用缩写src
    • destination 挂载到容器中的路径. 可以使用缩写dst或target
    • ro 配置项, 多个配置项之间由逗号分隔一般使用true或false
