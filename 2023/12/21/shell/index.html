Shell笔记

Donald Lee2023-12-212023-12-21

挂载：将磁盘的内容嵌入到根目录中。

文件系统命令

命令
描述
示例
ls
 调用不带参数的命令，则显示当前目录的内容 ls
显示指定目录的内容 ls /usr/lib/
pwd
 显示当前目录的路径 pwd
cd
 移动到指定相对或绝对路径的目录 cd /tmp
cd /usr/lib/
cd ..
mount
 将磁盘挂载到根文件系统，如果调用无参命令，则会显示所有已挂载磁盘的列表 mount
find
 查找文件或目录，其中第一个参数指定开始搜索的目录（允许使用正则表达式） find / -name vim
find /usr -name *vim*
grep
 通过文件内容查找文件 grep "/PATH" *
grep -Rn "/PATH" .
grep "/PATH" *. *

在上表中，Bash只能执行pwd 和 cd 命令， 这些称为内置命令。特殊实用程序执行表中的其他命令。如果Bash无法执行输入的命令，则会调用适当的实用程序。

波形符 ~ ： 使用波形符代替主目录绝对路径

Bash区分两种类型的参数：

  • 参数：参数是传递给程序的常规单词或字符
  • 选项：切换程序模式的参数。标准规定了选项格式，以破折号 - 开头或双破折号 – 开头的单词或字符

Windows和Unix都有隐藏文件和目录的概念，应用程序和操作系统根据自己的需要创建隐藏文件。这些文件存储配置和临时数据。

在Unix中，使用 ls -a 显示隐藏的对象

mount 命令列出的已挂载设备可能是本地的磁盘或分区，也可能是网络上的共享文件夹

下表调用不带参数的 mount 输出的结果

分区或目录
挂载点
文件系统类型
挂载参数
proc on /proc type proc (rw,nosuid,nodev,noexec,relatime)

区分路径类型：
绝对路径：以斜杠开头 /usr/include
相对路径：以目录开头 usr/include
.. 路径始终指向当前目录的父目录
. 指向当前目录

打印文件内容的两种方式： cat 、 echo “$(< filename)

每当调用命令时，Bash都会将调用的命令保存在命令历史记录中。可以使用向上或向下箭头浏览历史记录。
Ctrl+R 可以搜索所有命令历史记录。当输入字符时，Bash将向我们显示最后调用的以这些字符开头的命令，按下”Enter”来执行命令
history 不会保留输入然后删除的命令，只存储执行的命令。可以在命令前添加一个 # ，这样Bash会存储命令但不执行。在Bash中，**#** 符号代表注释，当解释器遇到注释时，它会忽略这一行。

当运行不带参数的 find ， 它会遍历当前目录的内容并打印，包括隐藏对象。
如果没有满足搜索条件的文件，find 不会打印任何输出
以下是 find 常用条件的格式

条件 描述 示例
-type f 仅搜索文件 find -type f
-type d 仅搜索目录 find -type d
-name pattern 搜索名称与区分大小写的全局模式匹配的文件或目录 find -name README
find -name READ*
find -name READ??
-iname pattern 搜索名称与不区分大小写的全局模式匹配的文件或目录 find -iname readme
-path pattern 搜索路径与区分大小写的全局模式匹配的文件或目录 find -path */doc/bash/*
-ipath pattern 搜索路径与不区分大小写的全局模式匹配的文件或目录 find -ipath */DOC/BASH/*
-a or -and 使用逻辑AND组合多个条件，如果找到的对象符合所有条件，则打印它 find -name README -a -path */doc/bash/*
-o or -or 使用逻辑OR组合多个条件，如果找到的对象至少符合其中一个条件，则打印它 find -name README -o -path */doc/bash/*
! or -not 使用逻辑NOT，如果找到的对象不符合条件，则打印它 find -not -name README
find ! -name README

全局模式：包含通配符的搜索查询。
Bash常用的三个通配符： * 、 ? 、 []
星号代表任意数量的字符; 问号表示任何类型的字符; 方括号表示特定位置的一组字符
在传入多个路径给 find 时， 在条件之间没有插入任何逻辑运算符，它默认插入AND

在 find 中可以使用转译括号对条件进行分组。例：

find / “(-path */doc/* -name README)” -o -name LICENSE

添加转移括号时，Bash会忽略他们，因此，Bash不会执行拓展，而是将所有搜索条件按原样传递给find

以下是 find 将操作应用与每个找到的对象

选项 描述 示例
-exec command {} \; 对每个找到的对象执行指定的指令 find -name README -type f -exec cp {} ~ \;
-exec command {} + 对找到的所有对象执行一次指定的命令，该命令接收所有这些对象作为输入 find -type d -exec cp -t ~ {} +
-delete 删除每个找到的目录，该命令仅删除空目录 find -name README -type f -delete

仅当调用的命令处理多个输入参数时才使用 + 号，如果命令只接收一个参数，它将处理低一个找到的对象并跳过其余的。

grep 在搜索时检查文件内容
find 和 grep 的选择:

  • find 按照名称, 路径或详细数据(元数据)搜索文件或目录
  • grep 对除文件内容以外一无所知时

grep 的参数

  • arg1: 用于搜索的字符串,始终将它用双引号引起来. 这样可以防止Bash扩展并保证实用程序接收到的字符串没有变化. 如果没有引号, Bash会将短语拆分为两个单独的参数. 这种将字符串拆分为单词的机制称为单词拆分
  • arg2: 文件的相对或绝对路径. 如果指定由空格分隔的文件列表, 该实用程序将处理所有的文件

grep 的选项

  • -n: 将行号添加到grep输出中
  • -R: 递归地遍历目录,并检查所有的文件(隐藏对象也会被检查; 在搜索时遵循符号链接)
  • -l: 显示包含短语的文件列表, 不需要每个文件中都出现该短语(短语至少出现一次)
  • -L: 显示没有该短语的文件列表

* .*: 获取隐藏对象

符号链接: 特殊类型的文件. 它包含指向另一个文件或目录的指针, 而不是数据

grep仅处理文本文件,因此它可以很好的处理源代码文件, 在实际工作中可以将它作为代码编辑器和IDE的附加组件

PATH: 环境变量存储实用程序和应用程序的路径列表. 该变量是一个命名的内存区域
env 调用不带参数的 env 会显示出目前使用的所有环境变量

$ 美元符号告诉Bash这是一个变量名, 当Bash在命令中遇到变量名时, 它会检查变量列表, 如果该名称存在, Bash会将变量值插入到命令中. 否则, 解释器会在那里放置一个空字符串

在PATH变量中, 冒号是用于路径列表的分隔

type 显示指定程序可执行文件的绝对路径
–help 打印可执行文件的简短帮助
info 打印可执行文件的文档系统

Bash

Bash功能:

  • I/O重定向
  • Pipelines(管道)
  • 逻辑运算符

Unix中的三种标准流

数字 名称 目的
0 标准输入流(stdin) 程序从该流接收输入数据. 默认情况下, 它来自键盘等输入设备
1 标准输出流(stdout) 程序在那里输出数据. 终端窗口默认打印该流
2 标准错误流(stderr) 程序在那里输出错误信息. 终端窗口默认打印该流
