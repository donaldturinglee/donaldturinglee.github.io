计算机网络

Donald Lee2024-01-072024-01-08

单播 : 一对一的通信, 例: 固定电话
广播 : 将消息从一台主机发送给与之相连的所有其他主机, 例: 电视信号
多播 : 将消息发给多个接收主机(与广播不同之处在于多播要限定某一组主机作为接收端), 例: 电视会议
任播 : 在特定的多台主机中选出一台作为接收端的一种通信方式(任播通信会从目标主机群中选择一台最符合网络条件的主机作为目标主机发送消息. 通常, 被选中的特定主机将返回一个单播信号, 随后发送端主机只会跟这台主机通信), 例: DNS

MAC地址和IP地址在标识一个通信主体时虽然都具有唯一性, 但只有IP地址具有层次性
MAC地址是真正负责最终通信的地址, 但在实际寻址过程中, IP地址却是不可缺少的

MAC寻址中参考的是地址转发表, 地址转发表中所记录的是实际的MAC地址本身.
IP寻址中所参考的是路由控制表, 路由控制表中所记录的是集中了之后的网络号

实际的地址转发表与路由控制表中能获取的信息并不是具体的目标地址, 而是该数据应该被发送出去的网卡信息

搭建网络的主要设备
网卡: 使计算机连网的设备
中继器: 从物理层上延长网络的设备
网桥/二层交换机: 从数据链路层上延长网络的设备
路由器/三层交换机: 从网络层转发分组数据的设备
4~7层交换机: 处理传输层以上各层网络传输的设备
网关: 转换协议的设备

带宽: 单位时间内传输的数据量的多少(bps)
吞吐量: 衡量带宽, 主机CPU, 网络拥堵程度, 保温中数据字段的占有份额(不包含报文首部, 只计算数据字段本身)等信息

中继器

  • 对减弱的信号进行放大和发送的设备
  • 通过物理层的连接延长网络
  • 即使在数据链路层出现某些错误, 中继器仍然能转发数据
  • 中继器无法改变传输速度

只单纯负责信号在0和1比特流之间的替换, 并不负责判断数据是否有错误. 同时, 它只负责将电信号转化为光信号, 因此不能在传输速度不同的媒介之间转发

网桥/二层交换机

  • 根据数据帧的内容转发数据给相邻的其他网络
  • 没有连接网段个数的限制
  • 基本上只用于连接相同类型的网络. 但是有时也可以连接传输速率不同的网络

数据链路的数据帧中存在一个FCS数据位, 用于校验数据是否正确送达目的地. 网桥还能通过地址自学机制和过滤功能控制网络流量
上述地址是指MAC地址, 硬件地址, 物理地址以及适配器地址, 也就是网络上针对网卡分配的具体地址
自学式网桥: 这类网桥会记住曾经通过自己转发的所有数据帧的MAC地址, 并保存到自己的内存表里

交换集线器中连接电缆的每个端口都能提供类似网桥的功能

路由器/三层交换机

  • 连接网络与网络的设备
  • 可以将分组报文发送给另一个目标路由器地址
  • 基本上可以连接任意两个数据链路

路由器是根据IP地址进行处理的. 由于路由器会分隔数据链路, 因此数据链路层的广播消息将无法继续传播

47层交换机
以TCP等协议的传输层及其上面的应用层为基础, 分析收发数据, 并对其进行特定的处理
为了能通过一个URL将前端访问分发到后台多个服务器上, 可以在这些服务器的前端加上一个负载均衡器
循环复用DNS: 通过对多个IP地址配置同一个名字, 每次查询到这个名字的客户得到其中的某一个地址, 从而使不同客户访问不同的服务器, 例: GitHub Page
宽带控制: 在网络比较拥堵时, 优先处理对及时性要求较高的通信请求(语音), 放缓处理稍有延迟也并无大碍的通信请求(邮件, 数据转发)
除上述应用场景之外, 47层交换机还能应用在广域网加速, 特殊应用访问加速以及防火墙等

网关

  • 负责协议的转换与数据的转发
  • 在同一种类型的协议之间转发数据叫做应用网关

网关不仅转发数据还负责对数据进行转换, 它通常会使用一个表示层或应用层网关, 在两个不能进行直接通信的协议之间进行翻译, 最后实现两者之间的通信, 例: 手机邮件与互联网邮件
在使用万维网时, 为了控制网络流量以及出于安全的考虑, 有时会使用代理服务器. 有了代理服务器, 客户端与服务器之间无需在网络层上直接通信, 而是从传输层到应用层对数据和访问进行各种控制和处理
代理服务器: 代替服务器提供服务的系统

边缘网络: 涉及接入层和汇聚层的网络
数据中心: 由大型服务器, 存储以及计算机网络构成. 面对高并发, 为了减少访问延迟, 会集合多个存储于一起, 通过连接高速网络, 以提高响应速度.

虚拟化技术: 不增减服务器, 存储设备, 网络等实际物理设备, 通过软件按量增减的一种机制. 通过这个机制实现按需分配, 按比例分配, 对外提供可靠的服务
云: 利用虚拟化技术, 根据使用者的情况动态调整必要资源的机制

TCP/IP一词泛指IP或ICMP, TCP或UDP, TELNET或FTP, 以及HTTP等网际协议族
TCP协议规范由RFC793号文档决定
IP协议规范由RFC279号文档决定

OSI参考模型注重”通信协议必要的功能是什么”
TCP/IP参考模型注重”在计算机上实现协议应该开发哪种程序”

TCP/IP分层
物理层, 数据链路层, 网络层, 传输层, 应用层

物理层
负责数据传输的硬件

数据链路层
利用以太网中的数据链路层进行通信, 把它当作让网卡起作用的”驱动程序”也可以. 驱动程序是在操作系统与硬件之间起桥梁的作用的软件

网络层
相当于OSI模型中的第三层网络层. 它使用IP协议. IP协议基于IP地址转发分包数据

TCP/IP分层中网络层和传输层的功能通常由操作系统提供. 尤其是路由器, 它必须得实现通过网络层转发分组数据包的功能.
连接互联网的所有主机跟路由器都必须实现IP的功能, 其他连接互联网的设备(如网桥, 中继器或集线器)就没有必要一定实现IP或TCP的功能

IP
分组交换的一种协议, 不具有重发机制, 属于非可靠性传输协议
ICMP
在IP数据包发送过程中发生异常, 给发送端发送一个发生异常的通知. 有时用于诊断网络的健康状况
ARP
从分组数据包的IP地址中解析出MAC地址的一种协议

传输层
最主要的功能是能够让应用程序之间实现通信

TCP
面向有连接的传输层协议, 可以保证两端通信主机之间的通信可以到达
TCP能够正确处理在传输中丢包, 无序传输等异常状况. 除此之外, TCP能够有效利用带宽, 缓解网络拥堵
在TCP中, 为了建立连接会导致网络流量的浪费. 此外, 为了提高网络的利用率, TCP协议中定义了各种各样复杂的规范, 因此不适用于视频会议(音频, 视频的数据量既定)等场合使用

UDP
面向无连接的传输层协议, 不会关注对端是否真的接收到传送过去的数据, 如果需要检查对端是否接收到分组数据包, 或者对端是否连接到网络, 需要在应用程序中实现.
UDP通常用于分组较少或多播, 广播通信以及视频通信等多媒体领域

应用层

浏览器与服务端之间通信所用的协议是HTTP(HyperText Transfer Protocol)
发送邮件的协议是SMTP(Simple Mail Tranfer Protocol)
邮件扩展协议MIME, 可以使邮件能发送声音, 图像等各式各样的信息, 修改邮件的文字大小, 颜色. 它属于OSI模型的表示层
文件传输协议FTP(File Transfer Protocol): 将保存在其他计算机硬盘上的文件转移到本地的硬盘上, 或将本地硬盘的文件转移到其他计算机硬盘上. 在传输过程中可以选择二进制或文本方式
FTP在进行文件传输时会建立两个TCP连接, 分别是发出传输请求时所要用到的控制连接与实际传输数据时所要用到的数据连接

远程登录协议TELNET与SSH: 登录到远程的计算机上

网络管理协议SNMP: 使用SNMP管理的主机, 网桥, 路由器等称为SNMP代理, 而进行传里的那一段称为管理器. SNMP正是管理器和代理所要用到的协议

包: 全能型述语
帧: 数据链路中包的单位
数据报: IP和UDP网络层以上的分层中包的单位
段: TCP数据流中的信息
消息: 应用协议中数据的单位

TCP首部

  • 源端口号和目标端口号(用于识别发送主机跟接收主机上的应用)
  • 序号(用以发送的包中哪部分是数据)
  • 校验和(用以判断数据是否被损坏)

IP首部

  • 接收端IP地址
  • 发送端IP地址

网络拓扑包括: 总线型, 环型, 星型, 网状型等
