Socket 初始化

两种类型的 socket：面向连接和无连接
两种协议：TCP 和 UDP

Sokcet 函数
socket(): 创建和初始化一个 socket
bind(): 绑定一个特定的 ip 和端口号
listen(): 用于在服务端监听新的连接
connect():用于在客户端设置远程地址和端口，在 TCP 的情况下，它还会建立一个连接
accept(): 用于在服务器上为传入的 TCP 连接创建一个新的 socket
send()和 recv(): 用于通过 socket 发送和接收数据
sendto()和 recvfrom(): 用于在没有绑定远程地址的情况下发送和接收数据
close(): 用于关闭 socket。在 TCP 的情况下，这会终止连接
shutdown(): 用于关闭 TCP 连接的一侧。它对于确保有序地关闭连接很有用
select(): 用于等待一个或多个 socket 上的事件
getnameinfo()和 getaddrinfo(): 提供了一种协议无关的处理主机名和地址的方式
setsockopt(): 用于更改一些 socket 选项
fcntl(): 用于获取和设置一些 socket 选项

为什么更倾向于使用 recv 和 send？
recv 和 send 能保持程序的可移植性

网络编程通常可以描述为四种类型之一：TCP 服务器、TCP 客户端、UDP 服务器、UDP 客户端

UDP

在 UNIX 下，socket 由标准文件描述符表示，这意味着可以使用任何标准 UNIX 文件 I/O 函数。
在 Unix 中，所有的文件描述符（包括 socket 描述符）都是小的非负整数，socket()函数返回一个 int, socket 返回一个负数表示错误

#define SOCKET int 这个宏可以在所有平台上将 socket 描述符存储为 SOCKET 类型
#define ISVALIDSOCKET(s) ((s) >= 0) 这个宏判断 socket 描述符是有效的还是无效的
#defien CLOSESOCKET(s) close(s) 这个宏用于关闭 socket
#define GETSOCKETERRNO() (errno) 这个宏用于当 socket 发生错误时，错误将被存储在线程全局的 errno 变量中

知识巩固：

  1. 什么 socket？
    答：socket 是表示系统之间通信链路的一端的抽象
  2. 什么是无连接协议？什么是面向连接协议？
    答：面向连接协议在较大的数据流上下文中发送数据包。而无连接协议则独立发送每个数据包，与之前或之后的任何数据包无关
  3. UDP 是无连接协议还是面向连接协议？
    答：无连接协议。每个消息都独立发送，与之前或之后的任何消息无关
  4. TCP 是无连接协议还是面向连接协议？
    答：面向连接。数据按照流的顺序发送和接收
  5. 通常有哪些类型的应用程序从使用 UDP 协议中获益呢？
    答：UDP 应用程序在牺牲可靠性的同时，获得更好的实时性能。它们还能够利用 IP 多播的优势。
  6. 通常有哪些类型的应用程序从使用 TCP 协议中获益呢？
    答：需要可靠的数据传输流的应用受益于 TCP 协议
  7. TCP 是否保证数据将成功传输？
    答： TCP 对可靠性提供了一些保证，但没有什么可以真正保证数据传输成功
  8. Berkely sockets 和 Winsock sockets 的主要区别是什么？
    答：头文件不同。socket 本身是用有符号整数表示的，而不是无符号整数。当 socket()或 accept()调用失败时，返回值也是不同的。Berkeley sockets 也是标准文件描述符，但在 Winsock 中并非总是如此。错误代码也是不同的，以不同的方式检索。
  9. bind()方法做了什么工作？
    答：bind()方法将 socket 与特定的本地网络地址和端口号关联起来。对于服务器，几乎总是需要使用它，而对于客户端通常不需要。
  10. accept()方法做了什么工作？
    答：accept()函数会阻塞程序执行，直到有一个新的 TCP 客户端连接。然后，它会返回此新连接的 socket
  11. 在 TCP 连接中，客户端还是服务器首先发送应用程序数据？
    答：客户端或服务器都可以首先发送数据。它们甚至可以同时发送数据。实际上，许多 C/S 协议(如 HTTP)的工作方式是先由客户端发送请求，然后由服务器发送响应。
